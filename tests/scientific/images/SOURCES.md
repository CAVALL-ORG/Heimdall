# Image fixture sources

Every image used by a test case is listed here with its origin,
license, and the expected canonical SMILES. If you add a new fixture,
add a row here AND a row in
[../../ketcher/image-to-smiles/image_sources.jsonl](../../ketcher/image-to-smiles/image_sources.jsonl).

The download script
[scripts/download_academic_images.ts](../scripts/download_academic_images.ts)
reads `image_sources.jsonl` and fetches into the right
`images/<category>/` subdirectory.

## Diverse fixtures (S14–S26)

55 fixtures (`D001`–`D055`) added 2026-05-15 to broaden chemistry-class
coverage of the image-to-SMILES suite. Authored by the
[`expand-image-to-smiles-suite.ts`](../../../outputs/expand-image-to-smiles-suite/expand-image-to-smiles-suite.ts)
driver: each compound's input SMILES is sourced from PubChem (CID cited
in the manifest `notes`), loaded into Ketcher, exported via
`exportSmiles`, and the round-tripped canonical is used as
`expected_canonical_smiles`. Every PNG is a Ketcher render of that
loaded molecule, so the image and the expected SMILES come from the
same canvas snapshot.

Image source: 100% Ketcher-rendered (no Wikimedia or academic fetches
in the diverse shard). This guarantees pixel-faithful fidelity to the
manifest SMILES. Drawing-style variation across publishers is covered
by the existing academic (S11) and Wikipedia (S12) shards.

Gap classes covered (see manifest `notes` for per-row class label):

To rebuild: `npx tsx outputs/expand-image-to-smiles-suite/expand-image-to-smiles-suite.ts`.
The driver is idempotent — it overwrites
`tests/scientific/images/diverse/<slug>.png` and rewrites
`outputs/expand-image-to-smiles-suite/manifest_additions.jsonl`
from scratch each run. To regenerate the manifest, restore
`tests/ketcher/image-to-smiles/manifest.jsonl` from HEAD and
re-`cat` the additions onto it.

## License hygiene

The download script writes one `<file>.LICENSE.txt` next to each
downloaded image with the citation and license line. Do not commit
images whose license you have not verified — add the URL to
`image_sources.jsonl` and let the script fetch it so the LICENSE.txt
gets generated alongside.

## Replacing a fixture with a journal figure

1. Update `image_sources.jsonl` to point at the journal's CDN URL.
2. Update `license` and `citation` fields with the journal's exact
   reuse terms (e.g. "CC BY 4.0, *PLOS ONE* DOI:10.1371/...").
3. Rerun `npx tsx scripts/download_academic_images.ts`.
4. If the journal forbids redistribution, fetch into a local-only
   directory (gitignored) and document the path locally — the test
   row remains in the manifest but the runner will mark the row
   `fixture_missing` on hosts that haven't fetched it.
